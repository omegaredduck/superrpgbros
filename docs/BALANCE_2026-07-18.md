# BALANCE PASS — Progression, Difficulty & Loot (2026-07-18, ?v=m9b)

Red's brief: **balance the game — drop rate of items vs the power they give as you
progress.** "Hard but fun. When you get the full legendary set it should be fairly
easy. Abilities telegraphed/fair. You should dodge but get hit sometimes and be
able to die — but the average player, locked in and focused, should succeed the
vast majority of the time. Each map ~10 min." All knobs below are **developer
tuning constants** (no in-game UI).

## The problem this fixes
The game had **no progression axis at all**. Every boss — the first corruption or
the final whale — rolled an *identical* drop table (T1-heavy, ~0.9%/roll legendary
tail), and mobs scaled only off the fuzzy character-level curve. So "loot vs power
as you progress" had nothing to hang on.

## Design (Red's locked calls)
- **Axis = campaign DEPTH** = number of regions already cleared
  (`ACCOUNT.cleared.length`, 0..19). Stamped on `scene.progressDepth` at realm
  create. Replaying an already-cleared region uses depth−1 (feels like that region,
  not the next). **Depth 0 == the pre-2026-07-18 flat game** (every multiplier is
  1.0), so fresh-save test suites are unaffected by construction.
- **Stay hard until the legendary set.** Difficulty ramps in lockstep with loot, so
  the campaign never gets easy. The set is auto-granted on the whale kill
  (`beatTheGame`) — that's the release valve, not drops.
- **No true one-shots.** `DATA.combat.maxSingleHitPct = 0.6` clamps any single hit
  to 60% of MAX HP in `Entities.hurtPlayer` (after class dmgTaken + boss-depth
  scaling + DEF). Death always takes ≥2 hits — even the ghost express (now a
  massive-but-survivable hit).
- **Separate projectile dials** — `DATA.tuning.projFreq.{mob,boss,player}` (default
  1.0 = no change). Scales player auto-fire, generic mob `shoot` cooldowns, and the
  core-boss radial/stream filler.

## The numbers (`DATA.progression`)
Loot per-tier depth factor (identity at depth 0): `lootShift = {1:0.93, 2:0.985,
3:1.055, 4:1.10, 5:1.105}`. Difficulty per-depth adders: `mobHpPerDepth 0.058,
mobDmgPerDepth 0.032, mobXpPerDepth 0.045, bossHpPerDepth 0.053, bossDmgPerDepth
0.028`. `maxDepth 19`.

### Loot — chance a drop is a given tier, by depth (Monte-Carlo validated)
```
depth │ white  green   blue  purple orange
   0  │ 51.7%  30.2%  12.9%   4.3%   0.9%   ← today's flat table (unchanged)
   6  │ 38.0%  31.3%  20.2%   8.7%   1.8%
  12  │ 24.7%  28.7%  28.0%  15.4%   3.3%
  19  │ 12.6%  21.9%  34.5%  25.5%   5.6%   ← the whale
```
End-of-campaign kit lands **blue/purple**; natural full-T5 set before the whale
≈ **0.3%** (the chase stays alive; the whale grant completes it).

### Difficulty — world threat by depth
```
depth │ mob HP  mob DMG  boss HP  boss DMG
   0  │ x1.00   x1.00    x1.00    x1.00
  12  │ x1.70   x1.38    x1.64    x1.34
  19  │ x2.10   x1.61    x2.01    x1.53
```
HP climbs faster than damage on purpose — longer/attritional fights (death from
stacked mistakes), not spikes.

## Where it lives (code)
- `sim.js` — pure helpers `depthMult(depth)`, `depthLootShift(entries, depth)`,
  `projFreq(kind)`. Identity at depth 0; loot shift returns the SAME array at depth
  0 so the RNG stream is untouched.
- `entities.js` — `spawnMob` composes `depthMult` into mob HP/XP/dmgMult (+ champion
  path); `spawnBoss` scales boss HP; `hurtPlayer` applies boss-depth damage + the
  no-one-shot cap; player fire + mob `shoot` cadence read `projFreq`.
- `scenes.js` — `progressDepth` stamped at realm create; `depthLootShift` applied at
  the three roll sites (boss chest, champion bounty, survival trial).
- `data.js` — `progression`, `tuning`, `combat.maxSingleHitPct`; `waves.mobLevelScale`
  retired to 0 (level→depth axis move).

## Retired
`waves.mobLevelScale 0.03 → 0` — the world-threat axis moved from character level to
campaign depth. `SIM.mobLevelMult` still composes (a nonzero value would stack).

## Testing
Full headless battery re-run. All 16 campaign-map verifies + core suites GREEN.
Four suites updated to the new design (ghost-express capped hit, grove level→depth
scaling, m3b permadeath forces the killing blow, m3c evolve isolated to depth 0).
Pre-existing/environmental non-greens (unchanged from baseline): `m1_suite`,
`m21_suite` (stale), `m23_cutscene` (a .wav cue can't load over file:// in the
headless sandbox; fine on the http server).

## Still TUNE-ME by playtest
Ramp magnitude, the ~10-min/map pacing (killQuota 150 + wave budget unchanged this
pass), and the projFreq dials (all 1.0 — levers staged, not yet pulled).
